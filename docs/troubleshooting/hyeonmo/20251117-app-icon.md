# Xcode 앱 아이콘 설정 및 Asset Catalog 구성

## 이슈 개요

- **증상**: Xcode 프로젝트에 앱 아이콘이 없어 빌드 시 기본 아이콘으로 표시됨.
- **영향 범위**: 모든 빌드 환경 (시뮬레이터 및 실제 디바이스).
- **감지 배경**: 앱 실행 시 홈 화면에 기본 흰색 아이콘만 표시되어 사용자 경험 저하.

## 진단 과정

1. Xcode Asset Catalog에서 `AppIcon` 항목이 비어있는 것을 확인.
2. 앱 아이콘은 1024x1024 크기의 PNG 파일이 필요하며, 알파 채널이 없어야 함을 파악.
3. 기존 디자인 파일(SVG 또는 다른 포맷)을 적절한 크기와 포맷으로 변환하는 작업 필요.

## 해결 방법

### 1. Icon Composer 사용

macOS 내장 Icon Composer를 사용하여 앱 아이콘 제작:

1. **디자인 준비**
   - Mentory 앱의 캐릭터 디자인 파일 준비
   - 1024x1024 크기에 맞게 구성

   <img src="https://github.com/user-attachments/assets/e244f4d0-a863-42ff-912f-bcd0fd10b022" alt="Icon Composer에서 아이콘 디자인" width="600" />
   
   <img src="https://github.com/user-attachments/assets/5f622587-a1e0-479d-a9e1-c56cd5e267ae" alt="다크 모드 아이콘" width="600" />

2. **Export 설정**
   - Platform: iOS, macOS 선택
   - Appearance: Dark/Light 모드 대응
   - Size: 1024pt
   - Light Angle: -45° (기본값)
   <img src="https://github.com/user-attachments/assets/b0be26cf-0d2b-47e6-b5f4-2f639dfca2d1" alt="Export 다이얼로그" width="600" />

3. **Asset Catalog에 추가**
   - Xcode에서 `Assets.xcassets` 열기
   - `AppIcon` 항목 선택
   - 1024x1024 슬롯에 export한 PNG 파일 드래그 앤 드롭

### 2. 검증

- Xcode에서 빌드 경고가 없는지 확인
- 시뮬레이터 및 실제 디바이스에서 홈 화면 아이콘이 정상적으로 표시되는지 테스트
- Dark Mode와 Light Mode 모두에서 아이콘 확인

## 회고 및 팁

- iOS 앱 아이콘은 **반드시 1024x1024 크기**여야 하며, 알파 채널(투명도)을 포함하면 안 됨.
- macOS Icon Composer를 사용하면 여러 크기를 자동으로 생성할 수 있어 편리함.
- 라운드 코너는 iOS가 자동으로 처리하므로, 원본 이미지는 사각형으로 제작.
- 다크 모드 대응을 위해 배경색이 너무 밝거나 어둡지 않은지 확인 필요.
